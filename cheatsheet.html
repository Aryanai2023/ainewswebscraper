High-yield decision rules
  • Less ops → managed services (RDS, DynamoDB, CloudFront).
  • Identity/permissions → IAM users, groups, roles, policies.
  • Web exploits (Layer 7) → AWS WAF. DDoS → AWS Shield.
  • Steady workloads → Savings Plans/RIs. Interruptible → Spot.
Rapid definitions

  • AZ: isolated data centers for fault tolerance.
  • Edge: CDN points of presence for low latency.
  • IAM: global identity and access management.

Domain 1: Cloud Concepts

What cloud is, why it’s used, and service models.

24%
Cloud computing
On-demand IT resources over the internet with pay-as-you-go pricing.
Six advantages
  • Trade capital expense for operating expense
  • Economies of scale
  • Elasticity (stop guessing capacity)
  • Speed and agility
  • No data center management
  • Go global quickly
Service models
Model You manage Example
IaaS OS, apps, configs Amazon EC2
PaaS Apps and data AWS Elastic Beanstalk
SaaS Usage/config only Managed business apps

Domain 2: Security & Compliance

Shared responsibility, identity, encryption, network controls.

30%
Shared responsibility
  • AWS: security of the cloud (facilities, hardware, core infrastructure).
  • You: security in the cloud (IAM, data, OS patching, network settings).
  • More managed/serverless = less ops responsibility for you.
IAM essentials
  • Users = people. Roles = assumed identities for services/apps.
  • Policies define permissions. Groups organize users.
  • Least privilege + MFA (especially root user).
Know these differences
  • Security Groups: instance-level firewall, stateful.
  • NACLs: subnet-level firewall, stateless.
  • AWS WAF: web app protection (Layer 7).
  • AWS Shield: DDoS protection.
  • AWS KMS: managed encryption keys.
  • AWS Artifact: compliance reports and agreements.

Domain 3: Technology & Services

Core services, infrastructure, monitoring and governance.

34%
Global infrastructure
  • Region: multiple AZs
  • AZ: isolation for availability
  • Edge: CloudFront delivery points
Compute & scaling
  • EC2: virtual servers
  • Lambda: serverless functions
  • ELB: distribute traffic
  • Auto Scaling: scale out/in
Storage & databases
  • S3: object storage, very high durability
  • EBS: block storage for EC2
  • RDS: managed relational
  • DynamoDB: managed NoSQL
Networking & delivery
  • VPC: private network
  • Route 53: DNS
  • CloudFront: CDN
  • Direct Connect: private link
Messaging & IaC
  • SQS: queue (decouple)
  • SNS: pub/sub notifications
  • CloudFormation: infrastructure as code
Monitoring & auditing
  • CloudWatch: metrics, logs, alarms
  • CloudTrail: API activity history
  • Trusted Advisor: best-practice checks

Domain 4: Billing, Pricing & Support

Pricing models, billing tools, support plan intent.

12%
EC2 pricing
  • On-Demand: short-term or unpredictable
  • Savings Plans / RIs: steady usage, discounts with commitment
  • Spot: cheapest, can be interrupted
Billing tools
  • Organizations: consolidated billing
  • Budgets: alerts on thresholds
  • Cost Explorer: analyze spend
  • TCO Calculator: estimate migration savings
Support plans
  • Basic: docs + forums
  • Developer: dev/testing support
  • Business: production, 24/7 support
  • Enterprise: mission-critical, TAM

Rapid Recall (Exam Traps)

Pick the right security service
  • Block web exploits → AWS WAF
  • DDoS protection → AWS Shield
  • Encryption keys → AWS KMS
  • Permissions → IAM policies / roles
Ops vs governance
  • Operational monitoring → CloudWatch
  • Audit trail of actions → CloudTrail
  • Cost/security/perf checks → Trusted Advisor
  • Decouple architecture → SQS + SNS
